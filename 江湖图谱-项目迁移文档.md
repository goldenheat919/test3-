## 一、项目概述

江湖图谱是一个基于 React + TypeScript + Vite 的武侠风格关系图谱可视化应用，用于以图谱形式展示武侠人物之间的三种关系：师徒关系、侠缘关系、结义关系。支持力导向布局、拖拽固定、增删改查、搜索筛选等交互功能。

### 核心功能

- 力导向自动布局（d3-force），节点自动排列
- 三种关系类型可视化：师徒（蓝色实线）、侠缘（粉色虚线）、结义（金色点线）
- 同一对角色多条关系自动曲线偏移，避免重叠
- 拖拽节点后自动固定位置，支持一键重置布局
- 孤立节点（无关系角色）自动标识并合理展示
- 角色增删改查（CRUD）
- 关系增删（选择角色 + 关系类型）
- 搜索角色（按姓名/头衔/描述模糊匹配）
- 关系类型筛选切换（图例点击显示/隐藏）
- 武侠古风 UI 风格（宣纸底色 + 毛笔字体 + 古典装饰）

## 五、数据模型

### Character（角色）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | `string` | 唯一标识（UUID） |
| `name` | `string` | 角色姓名 |
| `title` | `string` | 头衔/称号 |
| `generation` | `number?` | 师门辈分（第几代弟子，0 表示门外人物） |
| `description` | `string?` | 角色描述 |

### Relation（关系）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | `string` | 唯一标识（UUID） |
| `source` | `string` | 源角色 ID |
| `target` | `string` | 目标角色 ID |
| `type` | `RelationType` | `master_apprentice` \| `chivalrous` \| `sworn_brother` |
| `label` | `string?` | 关系描述（如"师父→弟子"） |

### RelationType（关系类型枚举）

```typescript
export const RelationType = {
  MASTER_APPRENTICE: 'master_apprentice',  // 师徒关系
  CHIVALROUS: 'chivalrous',                // 侠缘关系
  SWORN_BROTHER: 'sworn_brother',           // 结义关系
} as const;
```

## 六、本地部署步骤

### 环境要求

- Node.js >= 18
- npm >= 9

### 安装与运行

1. 将项目文件复制到本地目录
2. 打开终端，进入项目根目录
3. 执行 `npm install` 安装依赖
4. 执行 `npm run dev` 启动开发服务器
5. 浏览器打开 `http://localhost:5173` 查看效果
6. 执行 `npm run build` 构建生产版本（输出到 `dist/` 目录）

### 命令行参考

| 命令 | 说明 |
|------|------|
| `npm install` | 安装所有依赖 |
| `npm run dev` | 启动开发服务器（默认端口 5173） |
| `npm run build` | TypeScript 编译 + Vite 构建生产版本 |
| `npm run preview` | 预览构建后的生产版本 |

## 七、集成到现有 React 项目

如果您已有一个 React + Vite + Tailwind CSS 项目，可以将图谱组件集成到其中：

1. 安装依赖：`npm install @xyflow/react d3-force uuid`
2. 复制 `src/types/` 目录到您的项目
3. 复制 `src/data/` 目录到您的项目
4. 复制 `src/hooks/useForceLayout.ts` 到您的项目
5. 复制 `src/utils/graphTransform.ts` 到您的项目
6. 复制 `src/components/` 目录到您的项目
7. 在您的路由中引入 App 组件作为页面
8. 将 `src/index.css` 中的 `@theme` 变量合并到您的全局样式中

## 九、自定义数据

要使用自己的数据替换演示数据，修改 `src/data/index.ts` 文件：

- 将 `demoCharacters` 数组替换为您的角色数据（遵循 Character 接口）
- 将 `demoRelations` 数组替换为您的关系数据（遵循 Relation 接口）
- 关系类型使用 `RelationType.MASTER_APPRENTICE` / `CHIVALROUS` / `SWORN_BROTHER`
- 同一对角色可以有多条关系，系统会自动偏移显示
- 没有关系的角色会被自动标记为"孤立"并合理展示

### 数据示例

```typescript
import { RelationType } from './types';

const myCharacters = [
  { id: '1', name: '张三', title: '掌门', generation: 1, description: '某某派掌门' },
  { id: '2', name: '李四', title: '大弟子', generation: 2, description: '张三的大弟子' },
];

const myRelations = [
  { id: 'r1', source: '1', target: '2', type: RelationType.MASTER_APPRENTICE, label: '师父→弟子' },
];
```
